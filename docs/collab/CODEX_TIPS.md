# Codex Tips

## 当前项目判断

- `qianfamily` 当前是宗亲门户站，不是单纯的文化专题站，也不是已经接入真实业务的后台系统。
- 首页主逻辑是“文化认同 -> 谱系归属 -> 宗亲连接 -> 参与行动 -> 内容回流”，不要把首页误做成纯内容落地页或纯工具首页。
- `lib/portal-content.ts` 是门户导航、频道卡片和频道文案的主数据入口，改门户信息架构时先查这里。

## 当前路由判断

- `/genealogy/` 是家谱中心总入口，不要把寻根、续谱、接谱直接抬成并列顶级导航。
- `/heritage/` 是文化总频道，家训、吴越、人物和故事都应收在这个总频道下。
- `/events/` 和 `/gifts/` 目前仍是公开说明优先、真实报名和订单后补的结构。
- `/topics/[slug]/` 目前属于兼容专题入口，不是新的主导航方向。

## 当前工程判断

- 当前部署模型是 Next.js 静态导出加 Nginx 托管，默认不要把项目往 SSR、数据库依赖或 Node 常驻服务方向带。
- `output/` 里的截图和检查产物是本地验证材料，不是正式源码。
- 小程序还没有真实启动；任何 `apps/miniapp`、`miniprogram`、`taro`、`uni-app` 相关改动都应单独起任务。
