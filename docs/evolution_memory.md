# Evolution Memory

Last updated: 2026-03-10 21:20:05 CST

## 跨轮稳定结论（已验证）
- 首页必须持续承担“状态分流器”角色，不能退回为纯介绍页。
- 主线表达以“动作序列”优于“概念口号”：文化认同 → 谱系归属 → 宗亲连接 → 参与行动 → 内容回流。
- 对首次访客最有效的信息是“下一步做什么 + 需要准备什么 + 预计投入多少”。
- 低风险高收益策略是：在既有卡片与面板系统上增量加信息，而非引入新组件体系。

## 本轮新增沉淀（Iteration 4）
- 在首页分流卡中加入轻量预期提示（耗时/材料）可显著降低决策犹豫，且不破坏现有视觉系统。
- 家谱场景中，“退回原因 + 修正动作”比新增入口更有性价比，应优先做减摩擦。
- 活动频道采用“月内微时间轴”比长文说明更易形成连续行动。

## 本轮新增沉淀（Iteration 6）
- 家谱页中“准备项 ↔ 退回原因”一一映射比平铺两组信息更易执行，适合长期保留为提交前标准自检。
- 活动页中“72 小时回流清单”能把“回流”从口号变为可操作流程，优先保留时间节点表述（24h/48h/72h）。
- 对文化站点而言，新增模块应优先强化已有主线断点，而非扩展新频道或新入口位。

## 本轮新增沉淀（Iteration 7）
- 首页分流卡在“下一步去向”之前加入“可见产出”短句，可明显增强完成感，适合在高频入口长期保留。
- “投入提示 + 可见产出”成对出现时，比单独强调耗时更有说服力，能减少用户跳转后流失。

## 本轮新增沉淀（Iteration 8）
- 文化页中“读完后立刻执行”的三步桥接（联系人、地域收敛、三线索提交）能把抽象认同转为可启动动作，适合固定为文化频道收口模板。
- 跨页桥接文案需与家谱页退回经验统一，否则会产生“文化页建议与提交规则不一致”的信任损耗。

## 本轮新增沉淀（Iteration 10）
- 首页推荐卡若只给“做什么”仍有中断风险；补齐“怎么做（三步）+ 做完去哪”后，闭环显著更完整。
- 同一分区中多个入口若共享同一 CTA（如“进入入口”）会降低辨识度；差异化 CTA 文案可在不改布局的前提下提升动作确定性。
- 状态分流区保留一条“默认起步建议”可明显降低首次访客选择负担，且成本低、风险低，适合长期保留。

## 本轮新增沉淀（Iteration 11）
- 当 Playwright MCP 受本机浏览器会话冲突影响时，改用 `npx playwright screenshot` 可稳定产出跨端证据包，适合作为默认回退路径。
- 活动页在“看活动”与“去报名”之间插入“报名前 2 分钟自检”能有效降低临场信息不足导致的报名中断。
- 证据包建议固定四页（`/`、`/genealogy/`、`/events/`、`/heritage/`）双端（desktop/mobile）一次性生成，便于跨轮对比视觉收敛度。

## 本轮新增沉淀（Iteration 12）
- 首页推荐卡应提供“不适用时的跳过路径”，避免已有线索用户被默认推荐流程拖慢进入寻根。
- 在状态分流区直接展示“动作后回流”示例，比只在页尾公告强调回流更易触发执行。
- 首页新增回流卡应优先复用现有 `subtle-card` 体系，以信息增量替代组件增量，保持视觉收敛。
- 首页主文案若出现“不是…而是…”、“你可以把本站理解为…”这类内部论证句，会削弱用户动作感；应改写为“现在做什么、为什么、下一步去哪”。

## 本轮新增沉淀（Iteration 13）
- 首页一旦同时存在“状态分流”、“首次来站路径”、“给你的建议”三类解释块，就会出现重复指导，至少应删掉其中一类。
- 对首页而言，整块删除往往比逐句压缩更有效；用户只需要最短路径，不需要看到我们的推理过程。

## 本轮新增沉淀（Iteration 14）
- 门户页最容易失控的不是信息不足，而是“入口 + 解释 + 再解释”叠加；优先保留入口和准备项，重复解释块直接整段删除。
- 用户导向文案应稳定采用“现在做什么 / 需要准备什么 / 下一步去哪”三段式；一旦出现“频道、系统、规划、未来接入”字眼，通常应删或改。
- 活动页与文化页不适合同时堆“导读 + 节奏表 + 回流清单”三类说明；保留一个最直接的动作桥接即可。
- 礼品页最有效的减法不是少上商品，而是少讲频道逻辑，直接按使用场景组织选择路径。

## 本轮新增沉淀（Iteration 15）
- 专题页最容易显得“写给自己看”的位置是 Hero 首屏和 section-copy；优先把“本页是什么”删掉，改成“先看什么”。
- 深层内容页不宜同时保留“抽象判断 + 举例解释 + 再总结”三层表述；多数情况保留一层事实和一层结论就够。
- 历史与人物专题里，双句价值判断通常可以压成单句事实判断，不会损失可读性，反而更可信。
- 当数据源由 `lib/*-content.ts` 驱动时，优先在数据层减法，能同时收紧 `/heritage/*` 与 `/topics/*` 两套路由输出。

## 本轮新增沉淀（Iteration 16）
- 深层页面最隐蔽的冗余，往往不在业务数据，而在通用组件的默认标题和说明文案；需要定期回看组件层 copy。
- `WorkflowPage` 和 `DetailPage` 这类通用容器应固定使用动作导向标题，不能夹带“页面角色、系统能力、未来流程”类表述。
- 详情页重点卡的标题如果写成“页面作用、内容回流”，用户仍会感到在读产品设计；改成“参加前看什么、活动后看什么”更直接。
- FAQ 最有效的写法是“短问题 + 直接回答”，不需要借 FAQ 再解释一遍页面定位。

## 本轮新增沉淀（Iteration 17）
- 卡片 `summary` 最容易失控成“第二段导读”；一旦标题已经说明对象，摘要只需补“先看什么”或“适合什么场景”。
- 对入口卡而言，摘要最有效的结构是“动作 + 对象”；对公告卡和故事卡而言，摘要最有效的结构是“事实 + 范围”。
- 同一张卡片不应同时出现“动作、价值判断、主线解释”三层信息；多数情况保留一层即可。
- 数据层文案减法是高性价比动作：一次收紧可同时影响首页、频道页与复用列表，且几乎不带视觉风险。

## 本轮新增沉淀（Iteration 18）
- 当标题和摘要已清楚时，辅助字段只应承担三类信息：时间、场景、下一步；多出的解释通常都可以删。
- `effortHint / outcomeHint` 最容易写成“产品收益说明”；更有效的写法是短时间提示和明确结果，不要再加“可见产出、建议、理解主线”之类前缀。
- `meta` 不应再出现“站点更新、内容回流入口、文化意义先于商品展示”这种站点视角描述；应改成用户可感知的分类或场景。
- 辅助字段做减法后，卡片可读性提升明显，且不会影响版式，是适合持续迭代的低风险动作。

## 本轮新增沉淀（Iteration 19）
- 首页正文最容易失控成“策划说明书”；优先删除第二段解释，再检查每个区块是否只剩一条动作提示。
- 对首页来说，`section-copy` 最有效的长度通常是一句短句；超过两句时，往往已经开始解释站点方法论。
- “为何这样组织页面”不是用户需要的信息；首页正文只需要回答“先去哪、先看什么、再做什么”。
- 首页区块标题若带有“回流、闭环、主线”等内部表达，应优先改成用户能直接理解的入口标题。

## 本轮新增沉淀（Iteration 20）
- 标题也属于正文，不应借标题继续解释逻辑；标题应直接指向内容对象或入口对象。
- 如果一个区块的逻辑必须靠标题下的解释段落才能成立，说明标题本身不够直接，应改标题而不是补说明。
- 首页最稳定的写法是“内容名词 / 入口名词 + 卡片内容”，而不是“说明型标题 + 说明型段落 + 卡片”三层叠加。
- 用户反馈已明确：不再接受“内容即逻辑”的反向写法，后续应默认删除逻辑说明，除非缺少必要事实。

## 不建议重复尝试的方向
- 不建议在未验证转化前堆叠新入口或新频道入口位。
- 不建议脱离既有 `subtle-card / soft-panel / status-chip` 风格另起视觉分支。
- 不建议用抽象文化表述替代可执行步骤；抽象文案应作为补充而非主体。
- 不建议在同页新增两个以上“并列动作入口组”，避免行动选择过载。
- 不建议把截图证据产出绑定单一浏览器会话；自动化应优先准备可回退的截图通道（避免单点失败）。

## 页面结构长期共识
- 首页：首屏入口 + 状态分流 + 首次来站路径 + 频道深链。
- 家谱页：入口说明 + 提交准备 + 退回修正。
- 活动页：活动列表 + 参与路径 + 报名前自检。
- 文化页：专题入口 + 真实故事 + 读后动作。
- 礼品页：礼品精选 + 场景说明。
- 专题页：首屏导读 + 核心事实/案例 + 下一步。
- 工作流/详情页：首屏 + 三步重点 + 开始前确认 + 常见问题。

## 每轮优先检查清单
- 是否让首次访客在 10 秒内判断“我现在去哪里”。
- 是否减少提交反复和路径回跳。
- 是否让活动内容在“报名后”仍能回流为站内公共记忆。
- 是否维持视觉系统收敛（字体层级、间距、卡片、按钮一致）。
