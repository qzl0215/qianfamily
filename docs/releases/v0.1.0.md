# qianfamily v0.1.0

发布日期：2026-03-15

## 本版主题
- 完成 AI-Native 第一版结构升级。
- 完成 `src/` 迁移，并保持静态站公开路由不变。
- 建立规则层、记忆层、任务层、索引层与 AI 自进化脚本骨架。

## 重点变化
- 运行时代码迁入 `src/app`、`src/modules`、`src/shared`。
- 旧 `app / components / lib` 巨型结构已收口删除，不再作为主线继续扩张。
- `portal-home`、`genealogy`、`events`、`gifts`、`heritage`、`topics/*` 已建立模块边界与 `module.md`。
- 新增：
  - `docs/PROJECT_RULES.md`
  - `docs/ARCHITECTURE.md`
  - `docs/DEV_WORKFLOW.md`
  - `docs/AI_OPERATING_MODEL.md`
  - `memory/*`
  - `code_index/*`
  - `tasks/*`
  - `scripts/ai/*`

## 已解决的问题
- 拆除 `lib/portal-content.ts` 这类巨型内容总表。
- 首页与频道页开始从“大页 + 大内容”转向“路由壳 + 模块页 + section 组件”。
- AI 进入仓库后已有统一阅读顺序、任务模板、上下文构建与健康扫描入口。
- 多 agent 可围绕模块边界并行，而不必共享一个总表或总组件。

## 保持不变的对外契约
- 继续是单体 Next.js 静态导出站。
- 继续保留：
  - `/`
  - `/genealogy/`
  - `/heritage/`
  - `/events/`
  - `/gifts/`
  - `/topics/[slug]/`
- 继续使用 GitHub Actions + Nginx 生产发布。

## 验证结果
- `npm run typecheck`
- `npm run build`
- 生产部署工作流 `deploy` 已成功执行。

## 已知后续项
- `JiaxunTopicPage`、`GenealogyPage`、`EventsPage` 仍可继续拆成更细的 section 组件。
- GitHub Actions 已升级到 `actions/checkout@v5` 与 `actions/setup-node@v5`，用于消除 Node 20 action 弃用风险。
